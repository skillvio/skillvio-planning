# i18n — Çoklu Dil Stratejisi

## Hedef

Skillvio **dil-agnostik** bir altyapıyla başlar. İlk dil **Türkçe**, sonradan
**İngilizce, Almanca, Arapça** vs. eklenir — yeniden kodlama gerekmez.

## Temel Karar

**Locale-per-row pattern**: her translatable entity için sibling `_translations` tablosu.

```sql
-- ❌ Anti-pattern: locale alanları kolon olarak
-- tracks (id, title_tr, title_en, description_tr, description_en, …)

-- ✅ Skillvio kalıbı:
tracks              (id, code, level, color, …)
track_translations  (track_id, locale, title, subtitle, description)
```

### Neden?
- Yeni dil eklemek migration gerektirmez (sadece insert)
- Boş içerik tablo şemasını şişirmez
- Locale fallback (`tr` yok → `en` → default) kolay
- EAV anti-pattern'inden kaçınır (her dil için ayrı row, ama yapısal)

## Translatable Entityler

| Entity | Servis | `_translations` |
|--------|--------|------------------|

## Locale Çözümleme Zinciri

İstek geldiğinde **5 katmanlı fallback**:

```
1. URL prefix:   /en/discover  → 'en'
2. Cookie:        cs_locale=en
3. Accept-Language header
4. User profile:  preferred_locale veya ui_locale
5. Default:       'tr'
```

İçerik bulunamazsa fallback chain:
```
Requested locale → en → tr → first available
```

## URL Routing

İçerik dili URL'de görünür:

```
https://skillvio.io/tr/discover         ← TR içerik
https://skillvio.io/en/roadmap/python   ← EN içerik
https://skillvio.io/discover            ← cookie'den ya da default
```

SEO için:
- Her sayfada `<link rel="alternate" hreflang="tr|en|x-default">` etiketleri
- `sitemap.xml` her dil için ayrı

## Kullanıcı Tercihi

Profile'da iki ayrı ayar:

| Ayar | Anlamı |
|------|--------|
| `ui_locale` | Menü, butonlar, hata mesajları |
| `preferred_locale` | Lesson/roadmap içerik dili (varsayılan ui_locale) |

Örneğin Almanya'da yaşayan Türk kullanıcı:
- `ui_locale = de` (UI Almanca)
- `preferred_locale = tr` (içerik Türkçe)

## İçerik Çevirisi Pipeline

Yeni dil eklerken **AI taslak + admin onay** akışı:

```
1. Admin "Translate to EN" butonuna basar
2. AI service Gemini çağırır:
   - Prompt: "Translate the following lesson content from TR to EN.
     Keep technical terms (Lambda, S3, JWT) untranslated."
3. Çıktı `_translations(status='draft')` olarak yazılır
4. Admin manuel düzenler, `status='published'` yapar
5. Frontend yeni dilde göstermeye başlar
```

## Edge Case'ler

### Çevrilmez içerik
- AWS servis isimleri (`Lambda`, `S3`)
- Programlama anahtar kelimeleri (`return`, `async`)
- Kod örnekleri
- Sertifika kodları (`DVA-C02`, `AZ-104`)

Bunlar `payload_locale_neutral` JSONB kolonunda saklanır, hiçbir dile özel değildir.

### RTL desteği (Arapça için ileride)
- HTML `dir="rtl"` attribute
- CSS `logical properties` (`margin-inline-start` vs `margin-left`)
- Test edilecek breakpoint: tüm component'lerin RTL safe olduğu
- İlk faz: TR + EN (LTR) — RTL sonra

### Sayı ve tarih formatları
- `Intl.NumberFormat`, `Intl.DateTimeFormat` her yerde
- TR: 1.234,56 / 31.12.2026
- EN: 1,234.56 / 12/31/2026
